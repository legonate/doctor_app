# Doctor App

The **Doctor App** is a system developed as a university project. It is designed to create and manage objects representing users (patients, doctors, and admin). The system demonstrates **object-oriented programming** principles and data persistence/management. 

---

## Overview

The Doctor App manages the data and logic for a simulated healthcare appointment system. 
It allows for: 
- Doctor and patient object management
- Appointment creation, rescheduling, and cancellations
- Secure user authentication
- Role-based access control (Admin, Doctor, Patient)
- Organized architecture

---

## System Architecture

The project is structed using **object-orientated principles** where each class encapsulates its own data and behavior. This helps maintain modularity, readability, and reusability.
